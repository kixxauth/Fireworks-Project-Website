Be an archaeologist.

Where is cloud computing taking us?

  • We get to define it.

  • Approach the problem like an archaeologist.

  • Examine the artifacts of computing.

Batch Mode Processing

You passed your stack of cards to an operator who would run them as a batch and return the results.

commons.wikimedia.org/wiki/File:NASAComputerRoom7090.NARA.jpg

Time Sharing

Time sharing. Everyone got a small part of the mainframe using a terminal. Something like 30 people could work simultaneously.

commons.wikimedia.org/wiki/File:IBM-3279.jpg

Learn from the past

Time share mainframes had everything we are trying to do today.

  • File sharing.
  • Email.
  • Distributed computing.
  • Computing as a service.
  • Chat.
  • Social Games.
  • And did it all securely.

Cloud Computing

  • The World Wide Web is an application built on the Internet.
  • There are other applications built on the Internet along with the WWW including Email and Chat.
  • "Real" cloud computing is using the Internet as a time shared mainframe.

Getting Past the Transition Hack

  • The 9x email problem.
    andrewmcafee.org/2006/09/the_9x_email_problem/
  • Web applications are widely understood and they are currently the only tool we have in our toolbox.
  • As technologists it is our responsibility to bear the costs of moving past current web technology to build and adopt new tools.

REST

Caution: Do not ignore Representational State Transfer.

  • Roy Fielding Dissertation
    (www.ics.uci.edu/~fielding/pubs/dissertation/rest_arch_style.htm)
  • REST places constraints on system architecture, allowing it to flourish.
  • We need to have another look at REST in our new context.

Getting past the hacks.

We want a Framework!

Take a more differenter approach.

Requirements:

  • Must be in the web browser where nobody can take it away.
  • Must be modular.
  • Must be secure by mutual suspicion.
  • Must be community owned like the Apache HTTP server and Linux OS.
  • If these requirements are not met, somebody else like the Church of SteveJobsology, Willy Wonka and The Google Factory, or The Evil Empire, will tragically succeed with a "less open and more closed" framework.

Proposal

We need an open framework!

A rich client platform paired with distributed data services and a free T-Shirt.

Dynamic loading of JavaScript modules on the client.

  • CommonJS
  • Modules may be loaded from anywhere in the World, including the local device storage.
  • Make use of the file:// URL protocol.

Manage security between modules on the client.

  • Mutual suspicion.
  • Modules must be explicitly introduced.

Automatic dependency management on the client.

Allow the client to make cross domain HTTP requests.

  • Without the <script /> tag hack.
  • JSONRequest

Decoupled data services.

  • Use common protocols or provide client modules to interact with unique protocols.
  • Delegated authorization.
